## Installation

### Claude Code

```bash
claude mcp add reclaim -- npx -y @reclaimprotocol/mcp-server
```

Verify the server is connected:

```bash
claude mcp list
```

You should see `reclaim` with status `âś“ Connected`.

### Claude Desktop

Add to your `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "reclaim": {
      "command": "npx",
      "args": ["-y", "@reclaimprotocol/mcp-server"]
    }
  }
}
```

Restart Claude Desktop to apply changes.

## Available Tools

| Tool | Purpose |
|---|---|
| `issueCredentials` | Generate a fresh `appId` and `appSecret` pair and write them to `.env`. Creates credentials locally using a secp256k1 keypairâ€”no browser visit required. |
| `importCredentials` | Import existing credentials from the dashboard or another project. Validates the secret against the claimed appId before writing. |
| `searchProviders` | Search Reclaim providers by keyword. Returns proof volume and success rate signals when available. |
| `getProviderDetails` | Get full provider configuration and recent example proofs for a chosen `providerId`. |
| `checkAppStatus` | Check your app's sandbox/production status, sessions used, and quota remaining. |
| `verifySetup` | Health check that confirms your `.env`, SDK installation, and backend connectivity. Returns actionable suggestions for any issues. |
| `getIntegrationGuide` | Get framework-specific code scaffolds for Node.js, Next.js, React, or Python (FastAPI). |

## Configuration

The server accepts these environment variables:

| Variable | Default | Purpose |
|---|---|---|
| `RECLAIM_API_BASE` | `https://devapi.reclaimprotocol.org` | Backend API base URL |
| `RECLAIM_LOG_LEVEL` | `info` | Log verbosity: `debug`, `info`, `warn`, `error` |
| `RECLAIM_TIMEOUT_MS` | `15000` | HTTP timeout for backend calls |

Pass environment variables during installation:

```bash
# Claude Code
claude mcp add reclaim -e RECLAIM_LOG_LEVEL=debug -- npx -y @reclaimprotocol/mcp-server
```

```json
// Claude Desktop / Cursor / VS Code
{
  "mcpServers": {
    "reclaim": {
      "command": "npx",
      "args": ["-y", "@reclaimprotocol/mcp-server"],
      "env": {
        "RECLAIM_LOG_LEVEL": "debug"
      }
    }
  }
}
```

## Troubleshooting

| Issue | Solution |
|---|---|
| Agent says "the Reclaim tool list is empty" | Restart your MCP client. Claude Desktop requires a full app restart. |
| `Could not reach the Reclaim backend` | Check your network connection. Verify `curl https://devapi.reclaimprotocol.org/api/healthcheck` works. |
| `issueCredentials` won't create new credentials | Your `.env` already has credentials. Pass `force: true` to overwrite, or delete them manually. |
| `SANDBOX_LIMIT_NEAR` or `QUOTA_EXHAUSTED` | Your sandbox app has used its 100-session limit. Link the app at `dev.reclaimprotocol.org` to continue. |

## Security

- **Secrets stay local.** Credentials are generated locally and written to `.env`. The secret never leaves your machine.
- **Protected files.** `.env` is created with mode `0600` and automatically added to `.gitignore`.
- **Signature-based ownership.** When linking your app to your account, you prove ownership by signing a challengeâ€”the secret itself is never transmitted.
- **Audited cryptography.** Local keypair generation uses `@noble/secp256k1` + `@noble/hashes`, the same primitives used by ethers and viem.
